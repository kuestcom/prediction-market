## Required for production

| Variable | Why it matters | How to obtain it |
| --- | --- | --- |
| `KUEST_ADDRESS`<br/>`KUEST_API_KEY`<br/>`KUEST_API_SECRET`<br/>`KUEST_PASSPHRASE` | Powers proxy-wallet setup and authenticated trading requests. Without these, users cannot complete trading auth. | Generate credentials at [auth.kuest.com](https://auth.kuest.com/) by connecting your Polygon wallet. |
| `ADMIN_WALLETS` | Grants access to the admin dashboard. | Comma-separated list of wallet addresses for trusted operators. |
| `REOWN_APPKIT_PROJECT_ID` | Enables wallet logins through Reown (WalletConnect). | Copy from the [Reown dashboard](https://dashboard.reown.com/). |
| `BETTER_AUTH_SECRET` | Master secret for Better Auth sessions and encrypted trading credentials. | Generate a 32-character value in the [Better Auth docs](https://www.better-auth.com/docs/installation#set-environment-variables). |
| `CRON_SECRET` | Secures scheduled Vercel cron jobs. | Create any random string with at least 16 characters. |

<Callout type="warning">
  If `KUEST_*` credentials are missing or invalid, users can browse markets but will fail when creating trading auth/proxy wallet credentials.
</Callout>

---

## Recommended adjustments

- Branding, support links, analytics ID, fee recipient wallet, allowed market creators, LI.FI, and OpenRouter model/key are managed in **Admin > General Settings**.
- Market Context prompt/toggle is managed in **Admin > Market Context**.
