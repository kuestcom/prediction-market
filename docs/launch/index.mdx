## Getting started

Use [kuest.com/launch](https://kuest.com/launch) to begin the default launch flow.

Return to this docs section when you need details for:

- environment variables and production settings
- Cloudflare proxy configuration
- Vercel vs self-hosted infrastructure choices
- admin setup after the first deploy

- **Environment reference:** only the env vars that affect production launches.
- **Network proxy setup:** how to route Kuest infrastructure through your own domain.
- **Infra section:** deployment targets, storage-mode decisioning, and scheduler contract.
- **Admin guides:** one page for each admin section: General, Theme, Locales, Categories, Market Context, Affiliate, Events, and Users.
