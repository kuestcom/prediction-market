## Available targets

| Target | When to choose it | Runbook path |
| --- | --- | --- |
| **Vercel + Supabase** | Default hosted setup with the fastest path to launch | `infra/vercel/README.md` |
| **Docker Compose** | Single host, straightforward ops, local/prod parity | `infra/docker/README.md` |
| **Cloud Run** | Managed serverless on GCP | `infra/cloud-run/README.md` |
| **Fly.io** | Global app deployment with Fly runtime | `infra/fly/README.md` |
| **DigitalOcean App Platform** | Managed PaaS on DigitalOcean | `infra/digital-ocean/README.md` |
| **VPS (manual)** | Full control on Linux hosts (Droplet, EC2, Hetzner, etc.) | `infra/vps/README.md` |
| **Kubernetes** | Cluster-based deployment and ingress control | `infra/kubernetes/README.md` |
| **Terraform stacks** | Declarative provisioning for supported targets | `infra/terraform/README.md` |

---

## Target selection checklist

1. Choose whether you want the default Vercel + Supabase path or a self-hosted target.
2. Confirm your storage mode first (Supabase vs Postgres+S3).
3. Decide if you need managed platform ops or self-managed infrastructure.
4. Ensure your target supports, or pairs with, the scheduler model you need.
5. Lock deploy artifacts to immutable image references.

<Callout type="warning">
  External scheduler jobs for `/api/sync/*` are required only for non-Supabase deployments. Use the contract in [Infra Scheduler Contract](/docs/launch/infra-scheduler-contract).
</Callout>

<Callout type="info">
  Vercel + Supabase is the default path and uses the dedicated runbook at `infra/vercel/README.md`. Self-hosted targets should start from `infra/README.md` for shared environment variables and storage-mode requirements.
</Callout>
